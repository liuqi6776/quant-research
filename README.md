## 回测结果（更新）

### 均值回归策略回测（2024.01-2025.06）

使用**价格偏离20日均线**作为Max Pain偏离度的代理指标，进行完整回测：

| 指标 | 最优参数 | 买入持有 |
|------|----------|----------|
| **总收益** | **+49.20%** | +24.22% |
| **最大回撤** | **-7.22%** | - |
| **胜率** | **75.0%** | - |
| **交易次数** | **16次** | - |
| **Calmar比率** | **4.81** | - |

> ⚠️ **严重警告**：最优参数仅有**16次交易**分布在18个月中，存在严重**过拟合风险**。这不能证明策略有效。

### 默认参数回测

| 参数 | 结果 |
|------|------|
| 做多<-2%, 做空>+3%, 持有5天 | **-3.11%**（净收益） |
| 买入持有同期 | +24.22% |
| 超额收益 | **跑输27.33个百分点** |

**关键发现**：
1. 均值回归策略在牛市中严重跑输买入持有
2. 高胜率（57-75%）被低盈亏比拖累
3. 做空信号在趋势市中频繁触发导致亏损
4. 扣成本后单笔平均收益为负

## ⚠️ 已知局限性

### 局限1：同期偏差

`03_correlation_analysis.py` 使用同期数据（PCR/偏离度 vs 同一时间段涨跌），而非前瞻性数据（t时刻指标 vs t+1到t+N未来收益）。

### 局限2：样本量过小

`04_comprehensive_model.py` 基于单日8只ETF的截面快照（n=8），统计上完全无意义。

### 局限3：回测使用的是代理指标

`06_backtest.py` 用的是**价格偏离20日均线**代理Max Pain偏离度，**不是真正的Max Pain**。这验证的是"均值回归"概念，不是Max Pain指标本身。

### 局限4：历史Max Pain数据难以获取

A股ETF期权的历史Max Pain时间序列数据极难免费获取，本研究无法做真正的Max Pain前瞻性回测。

## 如何真正验证Max Pain策略

```python
# 需要的数据
1. 每日各ETF期权的完整T型报价（所有行权价的call/put持仓量）
2. 每日Max Pain计算结果
3. 连续1年+的时间序列

# 正确的回测逻辑
corr = np.corrcoef(deviation[t], return[t+1:t+N])[0,1]  # 前瞻性！
```

**推荐方案**：
- **方案A**：通过Wind/iFinD购买历史期权数据
- **方案B**：每日自行从中金所/上交所爬取并计算Max Pain
- **方案C**：使用有限的历史Max Pain点做简化验证（如本研究）
